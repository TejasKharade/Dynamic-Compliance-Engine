# Dynamic Compatibility & Configuration Compliance Engine

This project is a prototype compliance engine for enterprise endpoints. It converts compatibility rules from static documents into a structured graph model, then evaluates mock device inventory against those rules to find violations, explain root causes, and recommend remediation steps.

## What The Prototype Does

- Extracts compatibility rules from text or PDF documents using structured LLM output.
- Represents devices, BIOS versions, operating systems, drivers, agents, and firmware as graph nodes.
- Represents requirements, conflicts, recommendations, and warnings as graph relationships.
- Stores extracted graph data in Neo4j.
- Evaluates mock device inventory against compatibility rules using an LLM workflow.
- Produces LLM-generated compliance scores, findings, explanations, and remediation actions.

## Project Structure

```text
endpoint-compliance-engine/
  data/
    inventory/mock_devices.json        Mock endpoint inventory
    rules/compatibility_v1.txt         Sample compatibility rule document
  src/
    core/compliance_evaluator.py       Compliance scoring and finding logic
    database/cypher_queries.py         Neo4j Cypher query strings
    database/neo4j_client.py           Neo4j persistence client
    ingestion/text_extractor.py        LLM-based rule extraction
    schemas.py                         Pydantic data models
  tests/
    test_compliance_evaluator.py       Unit tests for evaluator behavior
    test_rules.py                      Manual extraction test script
  main.py                              LLM extraction + Neo4j + graph visualization flow
  run_compliance.py                    CLI entry point for inventory evaluation
```

## Setup

Install dependencies:

```bash
pip install -r requirements.txt
```

Create a `.env` file with:

```env
OPENAI_API_KEY=your_openai_key
NEO4J_URI=bolt://localhost:7687
NEO4J_USERNAME=neo4j
NEO4J_PASSWORD=your_neo4j_password
```

## Run LLM-Based Compliance Evaluation

This path extracts rules from `data/rules/compatibility_v1.txt` using the configured OpenAI model, then uses an LLM-powered LangGraph workflow to score devices, detect violations, explain findings, and recommend remediation.

```bash
python run_compliance.py --use-extracted-rules
```

## Run The Graph Ingestion Flow

This path extracts rules, pushes them into Neo4j, and opens a NetworkX/Matplotlib visualization.

```bash
python main.py
```

Neo4j must be running before this command is used.

## Run Tests

Run evaluator tests:

```bash
python -m unittest tests.test_compliance_evaluator
```

## Current Limitations

- The LLM extraction and compliance evaluation path depends on document quality and prompt accuracy.
- Compliance scoring is generated by the LLM, so outputs should be reviewed for high-stakes use.
- The dashboard and API layers are placeholders.
- Remediation actions are advisory and simulated.

## Recommended Next Enhancements

- Add a Streamlit dashboard for device-level findings.
- Add API endpoints for inventory evaluation.
- Add support for compound conditional rules.
- Add Neo4j read-back evaluation so rules can be evaluated directly from the stored graph.
